## Description

<!-- description:start -->

<p>Given two integers <strong>n</strong> and <strong>r</strong>, the task is to find the value of the <a href="https://www.geeksforgeeks.org/coefficient-in-binomial-expansion/" target="_blank" rel="noopener">binomial coefficient</a> <strong>nC<sub>r</sub></strong>. It can be defined as the coefficient of <strong>x<sup>r</sup></strong> in the expansion of <strong>(1 + x)<sup>n</sup></strong>, which gives the number of ways to choose <strong>r</strong> objects from a set of <strong>n</strong> objects without considering the order.</p>

<p>The binomial coefficient is calculated as: <strong>C(n,r) = n! / (r! * (n-r)!)</strong></p>

<p><strong>Note:</strong> If r is greater than n, return <strong>0</strong>.<br>
It is guaranteed that the value of <strong>nC<sub>r</sub></strong> will fit within a 32-bit integer.</p>

<!-- description:end -->

## üí° Solutions

### ü•â Approach 1: Brute Force (Using Factorials)

#### üìù Intuition

> - MDirectly apply the formula:
>   $$
>   C(n, r) = \frac{n!}{r! \, (n-r)!}
>   $$
> - Compute $n!$, $r!$, $(n-r)!$ and divide.
> - Simple and intuitive, works fine for small $n$ (here $n \le 100$).

#### üîç Algorithm

```pseudo
function nCr(n, r):
    if r > n: return 0
    return factorial(n) / (factorial(r) * factorial(n-r))
```

#### üíª Implementation

```cpp
// Brute force approach using factorials
class Solution {
public:
    // Compute factorial of n
    long long factorial(int n) {
        long long res = 1;
        for (int i = 2; i <= n; i++) {
            res *= i;
        }
        return res;
    }

    int nCr(int n, int r) {
        if (r > n) return 0;        // If r > n, return 0
        return factorial(n) / (factorial(r) * factorial(n - r));
    }
};
```

### ü•à Approach 2: Optimized Solution (Multiplicative Formula)

#### üìù Intuition

> - Avoid computing large factorials and divisions by using a multiplicative approach:
> - $$
> - C(n,r) = \frac{n \cdot (n-1) \cdot \dots \cdot (n-r+1)}{1 \cdot 2 \cdot \dots \cdot r}
> - $$
> - This reduces the risk of overflow and improves efficiency.
> - Still $O(r)$ time and $O(1)$ space.

#### üîç Algorithm

```pseudo
function nCrOptimized(n, r):
    if r > n: return 0
    if r > n-r: r = n-r   // Take advantage of symmetry
    result = 1
    for i = 1 to r:
        result = result * (n-i+1) / i
    return result
```

#### üíª Implementation

```cpp
// Optimized approach using multiplicative formula
class Solution {
public:
    int nCr(int n, int r) {
        if (r > n) return 0;          // r cannot exceed n
        if (r > n - r) r = n - r;    // Symmetry: C(n,r) == C(n,n-r)

        long long res = 1;
        for (int i = 1; i <= r; i++) {
            res = res * (n - i + 1) / i; // Multiply numerator and divide denominator
        }

        return (int)res;
    }
};
```

### ü•á Approach 3: Optimal Solution ‚≠ê - Pascal‚Äôs Triangle (Dynamic Programming)

#### üìù Intuition

> - Use the property of Pascal‚Äôs triangle:
> - $$
> - C(n,r) = C(n-1,r-1) + C(n-1,r)
> - $$
> - Build values iteratively in a 1D array to save space.
> - Time: $O(n \cdot r)$, Space: $O(r)$

#### üîç Algorithm

```pseudo
function nCrPascal(n, r):
    if r > n: return 0
    dp = array of size r+1 initialized to 0
    dp[0] = 1
    for i = 1 to n:
        for j = min(i,r) down to 1:
            dp[j] = dp[j] + dp[j-1]
    return dp[r]
```

#### üíª Implementation

```cpp
// Dynamic Programming approach using Pascal's triangle
class Solution {
public:
    int nCr(int n, int r) {
        if (r > n) return 0;

        vector<int> dp(r + 1, 0);
        dp[0] = 1; // Base case C(n,0) = 1

        for (int i = 1; i <= n; i++) {
            for (int j = min(i, r); j > 0; j--) {
                dp[j] = dp[j] + dp[j - 1]; // Pascal's formula
            }
        }

        return dp[r];
    }
};
```
